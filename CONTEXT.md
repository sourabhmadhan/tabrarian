## 1. What this project is

**Tabrarian** (formerly "Tab Tidy") is a Chrome Manifest V3 extension that
automatically organizes open tabs into Chrome's native named/colored tab
groups. Core promise: **nothing is ever closed automatically — tabs are only
organized**. Built for people who accumulate 50+ tabs and won't close them
for fear of losing something.

- Owner: Saurabh Madhan (sourabhmadhan@gmail.com), a software developer in
  Bangalore, India (LeetCode grinding, job hunting, AWS/Java/Spring work —
  the feature set reflects his real usage).
- Current version: **1.3.0**, ready for Chrome Web Store submission.
- Renamed from "Tab Tidy" → "Tabrarian" ("the librarian for your tabs")
  because the store already has "Tidy Tab Groups" and "TidyTabs – AI Tab
  Organizer" (name collision).

## 3. The categorization pipeline (categorizer.js)

Five layers, first match wins:

1. **Local check** — `isLocalHostname()`: localhost/127.0.0.1/0.0.0.0,
   `*.local|.localhost|.test|.internal`, private IPs (10.x, 192.168.x,
   172.16-31.x) → "Local" (green).
2. **Domain rules** — ~200 domains across categories: Dev, Cloud,
   Monitoring, Docs, AI, Video, Social, Shopping, News, Email, Work,
   Finance, Learning, Music, Reading, Travel, Government. Order in the
   CATEGORIES array matters (e.g. `irctc.co.in` hits Travel before the
   Government `.gov.in` TLD rule; `aws.amazon.com` hits Cloud before
   Shopping's `amazon.com`). Bare TLD entries work via suffix match:
   `"gov"` matches any `*.gov`; `"edu"`, `"ac.in"` → Learning.
3. **hostPatterns** — matches hostname *labels* (exact or `label-` prefix)
   to catch self-hosted internal tools on any company domain:
   grafana/kibana/prometheus/sentry/… → Monitoring;
   wiki/confluence/docs/kb/handbook/runbook → Docs;
   jenkins/gitlab/argocd/artifactory/sonarqube/backstage → Dev;
   jira/slack → Work. (e.g. `jenkins-prod.acme.com` → Dev.)
4. **Title keywords** — e.g. "tutorial" → Learning, "cart" → Shopping.
5. **Article heuristic** — `isArticleLike()`: `blog.` subdomain, or path
   containing `/blog|articles?|posts?|stor(y|ies)|essays?/` or `/YYYY/MM/`
   → "Reading".

Uncategorized tabs: clustered by domain (2+ same-site tabs = own group named
via `siteLabel()`), singletons → "Other". Pinned tabs and non-http(s)/file
URLs (chrome:// etc.) are never touched.

### Refinement pass (`refineGroups`)

- **Site splitting** (`SITE_SPLIT_MIN = 4`): if one site contributes 4+ tabs
  to a category with other sites present, it becomes its own group
  ("LinkedIn" out of Social).
- **URL_PRODUCTS** (path-aware splitting): hosts serving multiple products —
  `docs.google.com/document|spreadsheets|presentation|forms` → Google
  Docs/Sheets/Slides/Forms; `*.atlassian.net/wiki|browse|jira` →
  Confluence/Jira; `console.aws.amazon.com/<service>` → ~15 AWS services
  (EC2, S3, Lambda, CloudWatch, RDS, IAM, ECS, EKS, DynamoDB, SQS, SNS, VPC,
  CloudFormation, API Gateway, Secrets Manager); `console.cloud.google.com/*`
  → 10 GCP products. AWS region subdomains handled by suffix host match.
- **Localhost port splitting** (in `splitKey`): `localhost:3000` vs `:8080`
  become "Local :3000" / "Local :8080" (different ports = different apps).
- **Topic splitting** (`TOPIC_SPLIT_MIN = 8`): problem sites (leetcode,
  hackerrank, geeksforgeeks, codeforces, interviewbit, codechef) with 8+
  tabs sub-split by title keywords into "LeetCode · Trees/Graphs/DP/Linked
  Lists/Strings/Arrays" (TOPIC_RULES, first match wins — Trees before
  Linked Lists so "Convert Sorted List to BST" → Trees). One-tab topic
  groups fold back into the plain site group.
- Groups sorted biggest-first.

### Fuzzy matching to existing groups (`findMatchingGroup`)

Critical UX decision: **never dissolve or duplicate the user's hand-made
groups**. When grouping, a computed name is matched against existing group
titles: tokenize both (lowercase, split non-alphanumeric), expand aliases
(`TITLE_ALIASES`: lc→leetcode, gfg→geeksforgeeks, yt→youtube, so→
stackoverflow, gh→github…), then an existing group matches if **every one of
its tokens ⊆ computed name's tokens**. So user's "LC Trees" adopts computed
"LeetCode · Trees", "AI" adopts "AI", but "LC Revisit" / "LC To Do" are NOT
matched by plain "LeetCode" (their extra tokens signal curated intent).
Exact matches beat subset matches.

## 4. Topics mode (smartgroup.js) — local semantic clustering

User explicitly rejected API-based AI ("free, open source and safe for
all"). Implementation:

- Tokenize title (weight 1.0) + URL path slugs (weight 0.5); drop stopwords
  including **format words** (guide, review, tutorial, ultimate…) — these
  describe page *type* not *topic* and caused false clusters ("Tokyo Travel
  Guide" + "React Guide"); light stemmer (plural + -ing stripping).
- TF-IDF with **square-root-dampened IDF** (`sqrt(log(1 + N/df))`) — tuned
  so ONE shared topical word ("recipe") can bind two tabs. IDF computed
  over the current window only, so boilerplate ("- LeetCode" suffix ×27)
  self-cancels.
- Average-linkage agglomerative clustering, **threshold 0.10**, min cluster
  size 2. O(n³) fine for ≤200 tabs.
- Cluster label = top-2 tokens by summed weight ("Job · Software").
- Unclustered tabs fall back to the rules engine.
- Tuning was empirical: exp=0.5/th=0.10 chosen from a sweep; validated that
  a jobs cluster forms across linkedin/glassdoor/wellfound/cutshort, recipes
  cluster across 3 cooking sites, system-design articles cluster, while
  weather/music stay out and generic-word traps don't merge.
- Known ceiling: vocabulary overlap only — "car insurance" won't merge with
  "vehicle policy". Discussed upgrade path: Transformers.js + small local
  embedding model (~25 MB, Apache-2.0), same `buildTopicGroups` interface.

## 5. Background behaviors (background.js)

Message API from popup (`chrome.runtime.onMessage`): `groupAll`,
`groupOne`, `applyGroups`, `ungroupAll`, `collapseAll`, `closeDuplicates`.

- **Important architecture decision**: the popup computes the final grouping
  (including drag-and-drop overrides) and sends explicit specs
  `[{name, color, tabIds}]` via `applyGroups` — preview and applied result
  can never diverge.
- **Default scope**: only loose (ungrouped, unpinned) tabs are touched;
  `includeGrouped` flag (popup toggle "Reorganize existing groups", default
  off) enables full reshuffle.
- **Auto-group** (toggle, default off): `tabs.onUpdated` status=complete →
  categorize single tab → join fuzzy-matched existing group or create.
- **Focus mode** (toggle, default off): `tabs.onActivated` → expand the
  active tab's group, collapse all others. Purely visual; does NOT
  affect tab loading/discarding. Errors during tab drags ignored.
- **Memory saver** (toggle, default off): `chrome.alarms` every 5 min
  discards (`chrome.tabs.discard`) tabs that are: in a **collapsed** group,
  not active, not audible, not already discarded, `autoDiscardable !== false`,
  and unused for 10+ min (`tab.lastAccessed`). Discarded tabs stay in the
  strip and reload on click. Immediate sweep when toggled on. Constants:
  `MEMORY_SWEEP_MINUTES = 5`, `MEMORY_GRACE_MS = 10 min`.
- Duplicate closing: exact-URL dupes, keeps first occurrence, never closes
  the active tab.

## 6. Popup UI (popup.html/js/css)

- Modes: **Rules | ✦ Topics** segmented control (persisted as
  `groupingMode`). Switching clears drag overrides.
- Interactive preview: click tab → jump to it; hover × → close tab; search
  filters title+URL (auto-expands groups); per-group button says
  **"Join"** (with target name tooltip) when fuzzy match exists, else
  **"Group"**; group headers collapse/expand (chevron); "…and N more".
- **Drag & drop**: tab rows draggable onto group cards; moves stored in
  `overrides` Map (tabId → {name, color}), applied over computed groups;
  "Reset moves" link; stale overrides (closed tabs) self-clean.
- Header badge: "N loose · M in groups". Sleeping tabs shown dimmed/italic +
  "💤 N tabs sleeping" counter.
- Join-hint "→ joins "LC Trees"" shown on group headers.
- Custom instant tooltips via `data-tip` attribute + CSS `::after` (native
  title tooltips were too slow/truncated).
- Design: dark theme default + automatic light theme via
  `prefers-color-scheme`; Chrome tab-group palette as CSS vars (`--tg-*`);
  group cards have colored left border; iOS-style toggle switches; book-spine
  logo SVG matching the icon.
- Favicons via `chrome.runtime.getURL("/_favicon/") + ?pageUrl=…&size=16`
  (see security).

Settings in `chrome.storage.sync`: `autoGroup`, `regroupExisting`,
`focusMode`, `memorySaver` (booleans), `groupingMode` ("rules"|"topics").

## 7. Security audit (done)

- All dynamic UI rendered via `textContent`/`createElement` — no HTML
  injection from tab titles. Only `innerHTML` use is `= ""` clearing.
- No eval/remote code; no `externally_connectable` (pages can't message the
  extension); no content scripts; no host permissions.
- **Fixed vulnerability**: popup previously loaded `tab.favIconUrl`
  directly — a page-controlled URL → third-party requests on popup open
  (IP/timing leak, possible plain HTTP). Now uses Chrome's local `_favicon`
  cache (added `favicon` permission). Extension makes **zero network
  requests** overall.
- Nothing persisted except the 5 settings keys.

## 8. Testing approach

No test framework; logic verified via Node one-liners (categorizer/smartgroup
attach to `global.self`):

```bash
node -e "
global.self = global;
require('./categorizer.js');
const tabs = [{id:1, url:'https://github.com/x', title:'repo'}, ...];
for (const [n,g] of self.Tabrarian.buildGroups(tabs))
  console.log(n, g.color, g.tabs.map(t=>t.id));
"
```

Scenarios that must keep passing (regression set):
- 27 LeetCode tabs → topic sub-groups; "LC Trees" adoption; "LC Revisit" NOT adopted
- docs.google.com sheets vs docs split; AWS EC2/CloudWatch split; Local :3000/:8080
- grafana/wiki/jenkins/jira on a company domain → Monitoring/Docs/Dev/Work
- gov.in/irs.gov → Government; irctc.co.in stays Travel; mit.edu → Learning
- Topics mode: jobs-across-sites cluster, recipes cluster, no "Guide" false-merge
- chrome:// excluded; pinned excluded; duplicate detection keeps first + active

`node --check *.js` for syntax; `python3 -c "import json; json.load(open('manifest.json'))"`.

## 9. Competitive analysis (Tidy Tab Groups, 2k users, 4.3★)

They are **behavior-first** (focus mode, tab positioning, "invisible" UX),
with ZERO content categorization. We are **content-first**. We adopted their
best idea (focus mode); deliberately skipped tab auto-positioning and
home-tab anchor (fights Chrome's native behavior, high complexity).
Positioning: "Tabrarian both sorts your tabs intelligently AND keeps the
strip tidy as you work" — nobody does both.

## 11. Feature backlog (discussed, not built — user showed interest)

1. **Keyboard shortcut** (`chrome.commands`) to group without opening popup — trivial.
2. **Work/Gig profiles** — named rule-sets switched manually or by time of
   day (user works a job AND a side gig; wants different grouping contexts).
3. **Session snapshots** — save grouped window (names+URLs) to storage,
   restore later; directly addresses fear of losing tabs.
4. **Stale-tab review** — flag tabs unused 14+ days into a "Stale" group;
   export-then-close.
5. **Export groups to Markdown** (for Notion/Obsidian).
6. **Custom rules options page** — UI for domain→group mappings (good
   pre-launch candidate; currently requires editing categorizer.js).
7. Memory saver extension: optionally also sleep long-unused *loose* tabs
   (currently collapsed-groups only; ~2-line filter change).
8. Topics mode upgrade: Transformers.js local embeddings (~25 MB) for true
   semantic matching.
9. Icon alternates considered: leaning-book (concept B), tab-with-spines
   (concept C) — chosen A (four book spines on #2b3440) for both toolbar
   and store.

## 12. Tunable knobs (all in code, no build step)

| Knob | Location | Current | Effect |
|---|---|---|---|
| `SITE_SPLIT_MIN` | categorizer.js | 4 | tabs before a site splits out of its category |
| `TOPIC_SPLIT_MIN` | categorizer.js | 8 | tabs before problem-site topic split |
| `TOPIC_RULES` order | categorizer.js | Trees first | first-match topic priority |
| `TITLE_ALIASES` | categorizer.js | lc, gfg, yt… | user-shorthand expansion for group adoption |
| Topics `threshold` | smartgroup.js | 0.10 | lower = bigger/looser clusters |
| IDF dampening | smartgroup.js | sqrt | exponent on IDF; 1.0 = classic TF-IDF |
| `minClusterSize` | smartgroup.js | 2 | singleton clusters → rules fallback |
| `MEMORY_GRACE_MS` | background.js | 10 min | idle time before a tab can sleep |
| `MEMORY_SWEEP_MINUTES` | background.js | 5 | sweep frequency |
| Popup width / preview height | popup.css | 380px / 340px | layout |

## 13. Style/working conventions used

- Plain JS, no framework, no build; comments explain *why*, not *what*.
- Every logic change validated with a Node scenario test before shipping.
- Chrome tab-group palette: grey, blue, red, yellow, green, pink, purple,
  cyan, orange (the only colors `tabGroups.update` accepts).
- Privacy stance is a hard constraint: no network calls, no telemetry, no
  API keys — marketing and privacy policy promise it.
